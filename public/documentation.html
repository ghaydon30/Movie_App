Movies_App Documentation:

This project goal is to build the server-side of a movies web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Endpoints Description:

Return a list of ALL movies to the user.

  • Endpoint URL: /movies
  • Method Type: GET
  • Query Parameters: None
  • Expected Request Body: None
  • Response Data Format: JSON

Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user.

  • Endpoint URL: /movies/:title
  • Method Type: GET
  • Query Parameters: movie title (First letter Uppercase)
  • Expected Request Body: None
  • Response Data Format: JSON

Return data about a genre (description) by name/title (e.g., “Thriller”).

  • Endpoint URL: /movies/genre/:genreName
  • Method Type: GET
  • Query Parameters: Genre (first letter capitalized)
  • Expected Request Body: None
  • Response Data Format: JSON

Return data about a director (bio, birth year, death year) by name.

  • Endpoint URL: /movies/directors/:directorName
  • Method Type: GET
  • Query Parameters: Director name (first letters capitalized)
  • Expected Request Body: None
  • Response Data Format: JSON

Allow new users to register.

  • Endpoint URL: /users
  • Method Type: POST
  • Query Parameters: None
  • Expected Request Body: JSON object for new user
  • Response Data Format: JSON

Allow users to update their user info (username).

  • Endpoint URL: /users/:id
  • Method Type: PUT
  • Query Parameters: User id (as integer)
  • Expected Request Body: JSON Format "name":"newUserName"
  • Response Data Format: JSON

Allow users to add a movie to their list of favorites (showing only a text that a movie has been added).

  • Endpoint URL: /users/:id/:movieTitle
  • Method Type: PUT
  • Query Parameters: User id (as integer) and new favorite movie
  • Expected Request Body: None
  • Response Data Format: Text

Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed).

  • Endpoint URL: /users/:id/:movieTitle
  • Method Type: DELETE
  • Query Parameters: User id (as integer) and new favorite movie
  • Expected Request Body: None
  • Response Data Format: Text

Allow existing users to deregister (showing only a text that a user email has been removed).

  • Endpoint URL: /users/:id
  • Method Type: DELETE
  • Query Parameters: User id (as integer)
  • Expected Request Body: None
  • Response Data Format: Text
